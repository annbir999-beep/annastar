# AnnaStar Landing — правила работы с проектом

## Структура

- Один HTML-файл: `annastar-landing/index.html`
- Стили: `css/style.css`, `css/quiz.css`, `css/mobile.css`
- Скрипты: `js/` — каждый модуль отдельным файлом, без внешних библиотек
- Данные о работах: `data/works.json` + `js/works-data.js` (дублируются — см. ниже)

## Данные о работах

- Источник истины — `data/works.json`
- `js/works-data.js` — обязательный дубль для работы через `file://` (без сервера fetch не работает)
- **При любом изменении `works.json` — обновлять и `works-data.js`**
- Не хардкодить данные о работах в HTML или JS напрямую

## Оплата

- Ссылки на оплату берутся только из поля `payment_link` в `works.json`
- Платёжная система — ЮКасса
- Не вписывать ссылки вручную нигде, кроме `works.json`

## Дизайн

- Фон: `#1C1C1C` (тёмный), основной текст — светлый
- Акцентный цвет: золото `#B8860B` (в CSS-переменных: `--gold-mid: #b5804f`)
- Стиль — премиальный, без кричащих элементов
- Текст — философский, живой, без канцелярита и шаблонных фраз

## Изображения

- Работы: `images/works/` — формат JPG
- Мокапы и интерьеры: `images/mockups/`
- Фото художника: `images/artist/`

## Вёрстка

- Адаптивность обязательна: мобильные брейкпоинты в `css/mobile.css`
- При изменении любых стилей — проверять вид на мобильном (≤ 768px)
- Без внешних CSS-фреймворков и JS-библиотек

## Переводы (i18n)

- Переключатель RU/EN реализован в `js/i18n.js`
- Статический текст в HTML помечается атрибутом `data-i18n="ключ"`
- Плейсхолдеры — `data-i18n-ph="ключ"`
- Динамический контент (каталог, квиз) слушает событие `langchange`
- Добавляя новый текстовый блок — сразу добавлять перевод в `js/i18n.js` (оба языка)
- `js/i18n.js` должен подключаться **первым** в списке скриптов
